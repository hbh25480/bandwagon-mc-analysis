## 什么是搬瓦工 MC 机房（DC4 MCOM）

搬瓦工 MC 机房，全称 DC4 MCOM，机房编号 USCA\_4，物理位置在美国洛杉矶。MCOM 是 MultaCom 公司的简称，搬瓦工租用的是这家公司的机房和网络资源。MC 机房也是搬瓦工最早期的机房之一，从搬瓦工还在卖白菜价 VPS 的年代就在了。

**一句话定义**：搬瓦工 MC 机房是洛杉矶普通直连线路机房，三网回程均走公共线路，没有 CN2 或其他优化路由的保证，价格是搬瓦工体系里相对实惠的档位。

说"普通直连"不是贬义，是真实描述。它没有像 DC6、DC9 那样走中国电信 CN2 专线，也没有 CMIN2 这类移动优化线路，去程和回程都是正常公共路由。这就是为什么它便宜，也是为什么在某些场景下会拉胁——理解这点，后面选不选就很清楚了。

## 搬瓦工 MC 机房的真实线路情况

回程路由出机房之后走的是 MultaCom 自己的网络（AS35916），出 MultaCom 之后接 NTT 等国际骨干，再进国内三大运营商。

**联通**：联通对 MC 机房相对友好，AS4837 线路，延迟在正常范围内，白天速度尚可，算是三个运营商里表现最稳的。

**电信**：走 AS4134 公共骨干进来，延迟在 150ms 左右，晚高峰质量会波动，和走 CN2 优化线路的差距在高峰期会比较明显。

**移动**：延迟要高一些，CMI 线路进来再到大陆，整体表现三网中最弱。平均延迟 200ms 上下。

这是普通直连机房的典型表现，不算差，但和 DC6 CN2 GIA-E 放在一起比就差距拉开了。白天轻负载的时候差异不大，晚上 8 点到 11 点是真正拉开差距的时间。

顺便提一嘴：有些地区的电信用户接入 MC 机房时，实际路由会动态绕到 CN2 线路，但搬瓦工官方对此不作保证。如果你的需求依赖稳定的 CN2 回程，这种动态路由是靠不住的。

## FAQ

**Q：搬瓦工 MC 机房和 QNET 机房有什么区别？**

两者都是洛杉矶普通直连机房。MC 机房（DC4 MCOM）使用 MultaCom 的网络（AS35916），QNET 机房（DC2）使用 QuadraNet 的网络。线路表现相近，历史上联通用户对 QNET 评价略高一点，但两者差距不大，都属于普通直连这个等级。

**Q：我能买 KVM 套餐然后迁移到 DC6 CN2 GIA-E 机房吗？**

不能。机房迁移的范围取决于购买的套餐系列。KVM 套餐可以在几个普通直连机房之间切换（包括 MC），但进不了 DC6 CN2 GIA-E 机房——那个机房只有购买 CN2 GIA-E 套餐才能用。

**Q：搬瓦工 MC 机房的测试 IP 是多少？**

搬瓦工官方提供了 DC4 MCOM 机房的测试资源，购买前可以先 ping 一下看延迟情况，判断适不适合你的网络环境。

**Q：流量用完了怎么办？会不会扣额外费用？**

不会。搬瓦工的流量策略是月流量用完之后 VPS 自动暂停，等下个月流量重置后恢复，不产生额外账单。不过 VPS 暂停期间完全不可访问，如果需求比较大可以在后台升级到更高流量的套餐。

**Q：MC 机房的 IP 被封了怎么办？**

可以在 KiwiVM 后台通过切换机房来更换 IP，每次切换到不同机房 IP 都会变。但要注意，如果当前 IP 已经被封，切换机房功能本身可能也无法正常使用，这种情况需要联系官方客服处理。
